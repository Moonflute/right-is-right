# 📘 Right is Right (라이프스타일 타이머) 완벽 해부 가이드

프로그래밍을 처음 시작하시거나 웹(Web) 개발이 어떻게 이루어지는지 궁금하신 분들을 위해, 우리가 함께 만든 이 타이머 앱이 어떤 '부품'들로 이루어졌고 어떻게 생명력을 얻었는지 알기 쉽게 풀어낸 가이드북입니다.

---

## 🏗️ 1. 웹(Web) 앱을 구성하는 3가지 기본 언어
세상의 모든 웹사이트와 웹앱(우리가 만든 PWA 포함)은 기본적으로 3가지 언어로 구성되며, 이 앱 역시 예외가 아닙니다.

* **🧱 HTML (골격):** 화면에 버튼, 글씨, 네모난 상자들을 **"배치"**하는 언어입니다. 우리 앱에서는 "제목, 재생버튼, 목록"이라는 뼈대를 세웠습니다.
* **🎨 CSS (디자인/살점):** 뼈대에 색을 칠하고 크기를 조절하며 **"애니메이션"**을 넣는 언어입니다. 
   * *이 앱의 핵심:* "프로그레스 바가 오른쪽으로 차오른다, 깎여나간다" 라는 부드러운 애니메이션(`transition`과 `width` 속성)이 모두 순수 CSS를 통해 구현되었습니다. (우리는 `index.css` 파일에 이를 담았습니다)
* **🧠 JavaScript (모터/뇌):** 멈춰있는 뼈대와 살점에 생명을 불어넣어 **"동작"**하게 하는 프로그래밍 언어입니다.
   * 재생 버튼을 누르면 시간이 1초씩 올라간다. (`setInterval`)
   * 날짜가 바뀌면 하루 기록을 저장하고 0초로 리셋한다.
   * 위와 같은 논리적인 판단을 JS가 담당합니다.

## ⚙️ 2. 이 앱에 사용된 특별한 프레임워크 (신기술들)
맨바닥에서 3가지 기본 언어만으로 앱을 만들면 너무 힘들기 때문에, 현대 개발자들은 도구를 사용합니다.

### ① React (리액트) - UI 부품 조립 공장 🧩
Meta(구 페이스북)가 만든 가장 인기 있는 언어(라이브러리)입니다. 
* **왜 썼을까요?:** 화면을 하나의 통짜로 만들지 않고, 마치 레고 블록처럼 쪼개서 조립하기 위함입니다. 
* 우리 앱에서의 활용: `QuestItem`(타이머 1개)이라는 레고 블록을 딱 하나만 잘 만들어두고, 버튼을 누를 때마다 이걸 찍어내서 화면에 나열하는 방식입니다.
* *핵심 개념:* **"상태(State)"** - '점수', '흐른 시간' 같은 데이터가 변하면 리액트가 알아서 화면(UI)을 재빠르게 새로고침해 줍니다. 

### ② Vite (비트) - 초고속 건축 현장 ⚡
* **왜 썼을까요?:** 리액트로 짠 복잡한 부품들을 브라우저가 읽을 수 있는 HTML/CSS/JS로 순식간에 번역(빌드)해주는 도구입니다. 
* 여러분이 터미널에서 쳤던 `npm run dev`나 `npm run build`를 압도적인 속도로 처리해 주며, GitHub Pages로 코드를 뽑아낼 때 활약했습니다.

### ③ PWA (Progressive Web App) 기술 📱
* 분명 인터넷 브라우저로 접속한 웹사이트인데, **스마트폰의 앱 화면(홈 화면)에 아이콘이 다운로드**되고 전체화면으로 실행됩니다.
* 이를 위해 우리는 2개의 특수 파일을 만들었습니다.
  1. `manifest.json`: 앱 이름, 그리고 바탕화면에 깔릴 아이콘 이미지(화살표 아이콘)를 지정하는 명함.
  2. `sw.js (Service Worker)`: 핸드폰 시스템에게 "이건 앱으로 작동시켜도 좋아!"라고 승인(설치 허가)을 받는 일종의 백그라운드 관리자.

## 🏛️ 3. 앱의 설계 구조 (Architecture) 파악하기
이 프로젝트의 주요 소스 파일들은 `src/` (Source의 약자) 폴더 안에 모여있습니다.

* **`App.jsx` (총사령관):** 모든 데이터를 쥐고 있는 곳입니다. (퀘스트 목록, 전체 점수 계산, 1초마다 시간 올려주기 등)
* **`components/MasterGauge.jsx`:** 맨 위에 있는 전체 점수 환산 바(게이지)를 그려주는 부품입니다.
* **`components/QuestItem.jsx`:** 각각의 타이머 항목(수정, 리셋, 진행도 바 등)을 화면에 표시하는 부품입니다.
* **`components/HistoryModal.jsx`:** 자정이 지났을 때 지난 기록들을 팝업창으로 예쁘게 띄워줍니다.
* **`utils/audio.js`:** 딩동~ 삐빅! 소리를 만들어내는 브라우저 내장(Web Audio API) 신시사이저 도구 상자입니다.

> 💡 **데이터 저장의 비밀장소:** 
> 이 앱은 서버(Back-end) 컴퓨터가 없습니다! 그럼에도 껐다 켜도 시간이 유지되는 이유는, 사용자의 브라우저(크롬 폰) 내부에 있는 **"로컬 스토리지(Local Storage)"**라는 영구 보관함을 사용해서 데이터를 글씨 형태로 꾹꾹 눌러 담아뒀기 때문입니다.

## 🎓 4. 내가 이 앱을 직접 개발할 수 있으려면? (공부 로드맵)
지금 당장 코드를 외울 필요는 없습니다. 아래 순서대로 개념을 잡으시면 이 앱을 100% 이해하실 수 있습니다.

**[Level 1] 기초 공사 (HTML & CSS)**
* 1단계: HTML로 "div" 상자를 만들어 버튼과 글자를 배치해 보기
* 2단계: CSS로 `width: 50%;` 등 길이를 늘리고 색상을 입히며 막대기(Bar) 만들어 보기. (그리고 `transition`을 사용해 물 흐르듯 움직이게 만들기)

**[Level 2] 논리 부여 (자바스크립트 JS)**
* 3단계: JS를 사용해서 `if (조건)` 문과 `Math.max` 등으로 더하기/빼기 계산해 보기
* 4단계: `setInterval` 함수를 사용해 1초에 한 번씩 숫자가 올라가능 '기본 초시계(타이머)' 구현해 보기
* 5단계: 브라우저에 데이터를 저장했다가 꺼내보는 `localStorage` 기술 익히기

**[Level 3] 리액트 (React) 마스터하기**
* 6단계: 자바스크립트로 만든 것들을 **React 컴포넌트 단위**로 예쁘게 쪼개어 파일로 분리해 보기
* 7단계: 화면의 변화를 관리하는 마법 같은 함수 `useState`와 `useEffect`를 사용해서 타이머와 리스트 연동하기

이 안내서의 가이드를 따라 한 단계씩 미니 프로젝트(예: 텍스트만 있는 10초짜리 초시계 만들기 ➡️ 색칠해 보기 ➡️ 리액트로 옮겨보기)를 거치신다면, 어느새 이 앱을 완벽하게 해체하고 다시 조립하는 자신의 모습을 보게 되실 것입니다. 파이팅!
